[최솟값 찾기](https://www.acmicpc.net/problem/11003) 문제를 풀다가 재귀세그트리로 할 경우에 TLE가 나서 비재귀로 어떻게 더 효율적으로 짜는지 공부하고 정리하는 글이다. 앞으로는 해당 구현을 익히고 비재귀로 짜는 연습을 해야겠다. 정리해보고자 하는 세그트리는 **"최솟값"** 기준이다.

## 초기화

크기가 $N$ 인 배열을 `A` 라고 하고 세그트리를 `tree` 라고 할 경우에 초기화 방법이다.

```c++
for(int i=0; i<N; i++) tree[i+N] = A[i];
for(int i=N-1; i>0; i--) tree[i] = min(tree[i<<1], tree[i<<1|1]);
```

세그트리는 이진트리이기 때문에 리프노드의 시작이 인덱스 `N` 부터 시작한다. 따라서 배열값으로 먼저 초기화를 해준뒤에 리프노드의 직계부모 노드부터 시작해서 왼쪽 자식노드와 오른쪽 자식노드로 초기화해준다.



## 쿼리

구간 $[l, r)$ 의 최솟값을 찾으려 할 때의 쿼리이다.

```c++
int query(int l, int r){
  int ans = 2e9;
  for(l+=N,r+=N; l<r; l>>=1,r>>=1){
    if(l&1) ans = min(ans, tree[l++]);
    if(r&1) ans = min(ans, tree[--r]);
  }
  return ans;
}
```

위에서 말한 것처럼 리프노드의 시작이 `N` 이기 때문에 처음에 `l` 과 `r` 에 `N` 을 더해주고 리프노드부터 위로 올라가는 방식으로 쿼리값을 구한다. 이 때 $l$ 은 포함되고 $r$ 은 포함되지 않기 때문에 `l++` 와 `—r` 로 갈리게 된다. 이는 그림을 그려보면 쉽게 이해할 수 있다.
